## What This Package Does

MCP server (stdio transport) for the [Scoutify](https://github.com/matheusmarnt/scoutify) Laravel package. Fetches static docs from GitHub Pages and exposes 8 tools to AI assistants (Claude Code, Cursor, Codex, Gemini, Windsurf, Copilot, etc.) for documentation search and PHP code scaffolding. Prevents hallucination of removed/non-existent config keys.

## Key Design Decisions

### Static docs source (GitHub Pages)

Data source: `https://matheusmarnt.github.io/scoutify/llms.txt` + `llms-full.txt`. Generated by `docs-site/scripts/generate-llms-txt.mjs` in the scoutify repo on every `npm run build` (via `astro:build:done` hook). No API, no server — static files only.

`llms-full.txt` section format:
```
## <slug>

<page body>
```
**NO `---` separator before `## slug`** — removed during Phase A to avoid YAML frontmatter ambiguity. Parser in `search-docs.ts` and `get-page.ts` splits on `\n## `.

### Fetcher LRU cache

`src/fetcher.ts` — `Map<path, { body, etag, expiresAt }>`. 1h TTL per path. On expiry, sends `If-None-Match: <etag>` — on 304, refreshes `expiresAt` without new body. On non-2xx/304, throws `ScoutifyDocsUnavailable`. Free GitHub Pages stay within fetch budget (~1 req/hr per session).

### Validate snippet rules

11 regex rules in `src/scaffolds/validate-snippet.ts`. Rules mirror `ScoutifyServiceProvider::assertNoLegacyConfigKeys()` exactly. Reports line number via `code.slice(0, matchIndex).split('\n').length`. False-positive tested: `Scoutify::types()->register(...)` → 0 violations.

### Scaffold strategies — visibility rule

5 strategies in `scaffoldVisibilityRule`:
- `guests-allowed` → `visibleToGuests()->orWhenAuthenticated()->policy('view')`
- `auth-only` → `whenAuthenticated()->policy('view')`
- `spatie-role` → `whenAuthenticated()->role('<role>')`
- `spatie-permission` → `whenAuthenticated()->permission('<permission>')`
- `callback` → `whenAuthenticated()->using(fn ($user, $record) => /* TODO */)`

`HasGlobalSearchVisibility` is a **separate** opt-in contract from `GloballySearchable` — scaffold adds both to the class declaration.

### Build: bin entry + shebang

npm 10 auto-removes `package.json` bin entries with `./` prefix path (e.g., `./dist/server.js` gets removed silently). Fix: use `dist/server.js` (no `./`) as bin value. tsup `banner: { js: '#!/usr/bin/env node' }` injects shebang as first line of `dist/server.js`.

### npm publish auth

Granular Access Token type did NOT work for publishing new scoped packages (E404 on PUT despite PUT 200 on first attempt). Classic Automation token or re-auth via `npm set //registry.npmjs.org/:_authToken=<token>` required. 0.1.0 published but without bin (auto-removed). 0.1.1 fixed both bin + re-published with correct auth.

## Workflow Notes

- **No release-please** — manual `npm version patch` + `git tag` + `npm publish --access public`.
- **No worktree pattern** — repo is small enough to work on main branch directly for minor fixes. Use feature branches for larger changes.
- **Tests first** — TDD enforced. 37 tests in 7 files. Run `npm test` before every commit.
- **No external runtime deps** — only `@modelcontextprotocol/sdk`. All parsing is vanilla JS/TS.
